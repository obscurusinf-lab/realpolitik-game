## Конвенции

- Облачная сессия: работает в ветках `claude/*`
  - **Мелкие правки** (1 файл, явный изолированный фикс) — можно мержить самостоятельно после зелёного CI
  - **Крупные фичи** (несколько файлов, новая механика) — открывать PR, мержит локальная сессия после живого теста
- Локальная сессия: мержит крупные PR, разрешает конфликты, деплоит, тестирует UI на проде
- Vercel preview failure на feature-ветках — игнорировать (прод деплоится только с main)
- Всё важное — только через git: код, этот файл, комментарии к PR. Контекст чата между сессиями не передаётся

## CI

На каждый push/PR в main прогоняется `.github/workflows/ci.yml`:
- `frontend-build` — `npm run build` во frontend/ (ловит синтаксические и JSX-ошибки)
- `backend-syntax-check` — `node --check` на каждый файл в backend/src (ловит синтаксические ошибки)

Тестов и линтера в проекте нет — CI не проверяет игровую логику, только "код не сломан
синтаксически и билдится". Живое тестирование геймплея на проде остаётся за локальной
сессией. Зелёный CI — необходимое, но не достаточное условие для мержа: если облачная
сессия мержит свою ветку сама, она должна дождаться зелёного статуса перед мержем,
но это не отменяет ручную проверку на проде локальной сессией после.
