### Известные проблемы / не доделано
- Панель механик дублирует часть формул из бэкенда — при изменении балансовых порогов надо править оба места
- Telegram-мост не тестировался живьём
- «Утечка мозгов» (middle_class) — осознанно не трогали дальше, хотя теперь она таки причинна (компаундится в approval)
- **Категории действий заменены ПОЛНОСТЬЮ** — старые записи в истории партий (`gm_classification->>'action_type'`) остаются читаемыми на фронте (`ACTION_TYPE_LABEL` хранит и старые, и новые ключи), но партии, начатые ДО этого коммита, будут классифицироваться по новым правилам со следующего хода — не откатывать этот коммит частично
- Вкладка «Кремль» показывает ВСЕ категории домена как карточки (согласовано с игроком) — это значит некоторая избыточность в подсказках (например «Указы» — 13 карточек на одном экране, скроллится)
- exposure_risk penalty (диалог/стабильность −3/−1 при раскрытии) — фиксированные числа, не завязаны на severity операции, можно доработать
- Тестировал локально через мок callClaudeApi (ключа Anthropic нет в облачном окружении) — реальную классификацию ИИ по новым категориям не проверял, только структуру/расчёты/UI. Стоит последить за первыми реальными партиями — не путает ли ИИ похожие категории (например `mil_tactical` vs `mil_operational_offensive`)

## Конвенции

- Облачная сессия: работает в ветках `claude/*`, НЕ мержит в main сама
- Локальная сессия: мержит, разрешает конфликты, деплоит, тестирует UI на проде
- Vercel preview failure на feature-ветках — игнорировать (прод деплоится только с main)
- Всё важное — только через git: код, этот файл, комментарии к PR. Контекст чата между сессиями не передаётся

## CI

На каждый push/PR в main прогоняется `.github/workflows/ci.yml`:
- `frontend-build` — `npm run build` во frontend/ (ловит синтаксические и JSX-ошибки)
- `backend-syntax-check` — `node --check` на каждый файл в backend/src (ловит синтаксические ошибки)

Тестов и линтера в проекте нет — CI не проверяет игровую логику, только "код не сломан
синтаксически и билдится". Живое тестирование геймплея на проде остаётся за локальной
сессией. Зелёный CI — необходимое, но не достаточное условие для мержа: если облачная
сессия мержит свою ветку сама, она должна дождаться зелёного статуса перед мержем,
но это не отменяет ручную проверку на проде локальной сессией после.
