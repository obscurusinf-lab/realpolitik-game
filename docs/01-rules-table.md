# Таблица правил влияния (Rules Table)

Это единственный источник правды о том, как ход игрока меняет числа.
ИИ-геймместер **не придумывает** величины изменений — он классифицирует ход
по этой таблице, алгоритм считает дельты, ИИ только описывает результат словами.

---

## 1. Структура показателей (stats)

| Показатель | Диапазон | Смысл |
|---|---|---|
| economy | 0–100 | Состояние экономики: бюджет, рост, инфляция |
| military | 0–100 | Боеспособность, инициатива на фронте/в регионе |
| stability | 0–100 | Внутренняя устойчивость: протесты, элиты, регионы |
| diplomacy | 0–100 | Вес на международной арене, союзы |
| approval | 0–100 | Поддержка населения |

Каждый показатель имеет **инерцию**: за один ход не может измениться больше
чем на величину из колонки `max_delta_per_turn` (см. ниже) — это и страховка
от взрыва баланса, и реализм (экономика не падает на 30 пунктов за один указ).

```
max_delta_per_turn = {
  economy:    6,
  military:   6,
  stability:  5,
  diplomacy:  4,
  approval:   5
}
```

---

## 2. Категории ходов (action_type)

Каждый текстовый ввод игрока классифицируется ИИ в одну из категорий.
У категории — диапазон влияния на каждый показатель (алгоритм выбирает
конкретное значение в диапазоне на основе severity, см. п.3).

| Категория | economy | military | stability | diplomacy | approval | Пример хода |
|---|---|---|---|---|---|---|
| `military_offensive` | -2..0 | +1..+5 | -2..0 | -3..0 | -1..+2 | Удар по объектам, наступление |
| `military_defensive` | -1..0 | 0..+3 | +1..+3 | 0..+1 | +1..+3 | Усиление ПВО, оборона |
| `diplomacy_outreach` | 0..+2 | 0 | 0..+1 | +2..+5 | 0..+1 | Переговоры, новый альянс |
| `diplomacy_confrontation` | -2..0 | 0 | -1..0 | -4..-1 | -1..+2 | Ультиматум, разрыв отношений |
| `economic_stimulus` | +1..+4 | 0 | +1..+2 | 0 | +1..+3 | Льготы, инвестиции |
| `economic_austerity` | +2..+5 (долгосрочно -1..0 краткосрочно)* | 0 | -3..-1 | 0 | -3..-1 | Сокращение расходов, налоги вверх |
| `domestic_repression` | 0 | 0..+1 | +1..+3 (краткосрочно) / -2..0 (долгосрочно)* | -2..0 | -3..-1 | Подавление протеста, цензура |
| `domestic_liberalization` | 0..+1 | 0 | -1..+2 | +1..+2 | -1..+3 | Снятие ограничений, реформы |
| `info_narrative` | 0 | 0 | 0..+2 | -1..+2 | +1..+3 | Информационная кампания |
| `intelligence_covert` | 0 | +1..+3 | 0 | -2..0 (если раскрыто) | 0 | Тайная операция |
| `peace_initiative` | +1..+2 | -1..0 | +1..+2 | +2..+4 | +1..+3 (если не сорвано) | Инициатива перемирия |
| `null_action` | -1..0 | -1..0 | 0 | 0 | -1..0 | Игрок ничего не делает / бездействие |

\* Для категорий с временным расщеплением эффекта — система хранит
`delayed_effects[]` (см. п.5), которые применяются через N ходов.

---

## 3. Severity (модификатор силы хода)

ИИ оценивает не только категорию, но и **масштаб** хода по шкале 1–3:

| Severity | Множитель к диапазону | Описание |
|---|---|---|
| 1 — minor | 0.3–0.5 | Локальная мера, символический жест |
| 2 — standard | 0.6–0.85 | Обычное решение в рамках категории |
| 3 — major | 0.85–1.0 | Крупное, рискованное, прецедентное решение |

Конкретное число внутри диапазона категории — `round(min + (max-min) * multiplier)`,
multiplier берётся из середины диапазона severity (детерминированно по seed хода,
не по "настроению" ИИ — важно для воспроизводимости и честности лидерборда).

---

## 4. Влияние на отношения (relations)

Отдельная подтаблица — ход может задевать конкретную страну/блок напрямую.
ИИ обязан вернуть `affected_relations: [{country, delta}]`, где `delta` ограничен:

```
max_relation_delta_per_turn = ±8  (для прямого взаимодействия со страной)
max_relation_delta_per_turn = ±3  (для побочного эффекта, страна не цель хода)
```

Правило **спилловера**: если отношения со страной A меняются на X,
союзники/противники A (по `relations_graph`, см. countries.json) получают
`X * 0.25` в ту же сторону, противники A — `X * -0.15`.

---

## 5. Отложенные эффекты (delayed_effects)

Некоторые категории (`economic_austerity`, `domestic_repression`,
`peace_initiative` если нарушено) пишут в очередь:

```json
{
  "trigger_turn": 12,
  "effect": { "stability": -2 },
  "reason": "Отложенный эффект жёсткой экономии: накопленное недовольство"
}
```

Алгоритм на каждом ходу проверяет очередь и применяет созревшие эффекты
**до** обработки нового хода игрока — это и есть механика "мир реагирует
не только на твой текущий ход, но и на накопленную историю".

---

## 6. Ходы соперников / реакция мира (world_turn)

Раз в N ходов (или по триггеру — например diplomacy < 30) система генерирует
"ход мира": другая страна-актор совершает действие по тем же категориям,
но от своего имени. Логика:

1. Алгоритм выбирает актора (вес = f(relations_value, текущие hotspots))
2. ИИ генерирует конкретное действие в рамках категории + severity
3. Действие проходит через ту же таблицу дельт, но зеркально на игрока

Это даёт честную процедуру вместо "ИИ решает что хочет" — категории и диапазоны
те же самые, что и для игрока.

---

## 7. Score для лидерборда

Итоговый счёт партии — не сумма stats (это даёт вырожденные стратегии типа
"качай только approval"). Формула:

```
score = stability * 0.25
      + diplomacy * 0.2
      + economy   * 0.2
      + approval  * 0.15
      + military  * 0.15
      + survival_turns * 0.5          -- бонус за длительность без коллапса
      - collapse_penalty               -- если stability/approval < 15 в любой момент: -20
```

Хранится как `score_breakdown` JSON в `leaderboard_snap` — чтобы можно было
показать не только число, но и "за счёт чего" игрок набрал балл.
